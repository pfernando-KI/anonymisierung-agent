# Sicherheitsrichtlinie

## Sicherheitslücken melden

**Bitte melde Sicherheitsprobleme nicht über öffentliche Issues.** Öffentliche
Issues machen eine Schwachstelle für alle sichtbar, bevor ein Fix bereitsteht.

Bitte nutze stattdessen die private Reporting-Funktion von GitHub:

> **Repository → Security → Advisories → Report a vulnerability**

Anleitung von GitHub dazu:
https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing-information-about-vulnerabilities/privately-reporting-a-security-vulnerability

So bleibt deine Meldung vertraulich, bis das Problem behoben ist.

## Was du erwarten kannst

- **Erste Rückmeldung:** innerhalb von **7 Tagen** nach Eingang der Meldung.
- **Fix-Ziel:** innerhalb von **30 Tagen** nach Bestätigung der Schwachstelle.

Wenn ein Fix mehr Zeit braucht, informieren wir dich über den Stand.

## Unterstützte Versionen

Sicherheitsupdates werden aktuell nur für die jeweils neueste Version bereitgestellt.

| Version | Unterstützt |
|---------|-------------|
| `main` / `v1.0.0` | ✅ |
| ältere Stände | ❌ |
